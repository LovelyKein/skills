# 基金操作日志

> 由AI维护，每次交易后由用户告知并同步更新。
> 按日期倒序排列，交易/转换/分红等均记入。

## 基金操作规则参考

### 交易时间（A股基金）

| 操作时间                | 成交净值             | 举例                       |
| ----------------------- | -------------------- | -------------------------- |
| 交易日 **15:00前**      | **T日**净值          | 周一14:00买入 → 按周一净值 |
| 交易日 **15:00后**      | **T+1日**净值        | 周一15:10买入 → 按周二净值 |
| 非交易日（周末/节假日） | **下一个交易日**净值 | 周六买入 → 按周一净值      |

- A股交易日：周一至周五（法定节假日休市）
- 基金每天只有一个净值，15:00收市后计算，通常18:00后公布

### 买入（申购）

- **C类份额**：免申购费，按上面时间规则确认净值
- **A类份额**：有申购费，蚂蚁上通常打1折
- 最低买入金额：蚂蚁/天天通常 **10元起**（具体看产品）

### 卖出（赎回）

**持有时间从确认日开始算，不是从买入日算。**

> 确认日 = 买入日（T日）的下一个交易日（T+1）

| 持有时间 | C类赎回费            | 说明                                  |
| -------- | -------------------- | ------------------------------------- |
| < 7天    | **1.50%**（惩罚性）  | 持仓中任何基金7天内卖出都会扣这个费率 |
| 7-30天   | 通常 0.50%~0.75%     | 具体看各基金合同                      |
| ≥ 30天   | C类通常 **免赎回费** | 持仓中C类基金大多在这个规则内         |

**赎回资金到账**：T+2 ~ T+4 个交易日到余额宝/银行卡。

### 转换

- **转换 = 赎回A + 申购B** 一步完成，通常以同一确认日的净值计算（以实际确认记录为准）
- 节省时间：不经过赎回到账等待期，无缝切换
- 持有时间计算（分两种情况）：
  - **同公司转换**：转出基金和转入基金的持有时间延续计算（费率按持有时间阶梯计算）
  - **跨TA转换**（不同基金公司之间，常用的模式）：转出基金计算其赎回费，转入基金持有期从确认日重新开始计算
- 适用场景：同一基金公司旗下产品之间（不同公司的产品可能不支持互转）

### 净值说明

- 基金净值**盘中没有实时值**（ETF除外），蚂蚁/天天上的"估算净值"仅供参考
- 指数型基金估算误差通常较小
- 主动管理型基金估算可能与实际有明显偏差

## 🔍 自检机制

每次输出简报/更新数据前，按以下规则逐一检查，发现问题立即告知用户。

### 检查项1：操作日志完整性检查

- 每次对话开始时，检查 `fund-holdings.md` 中的基金清单是否全部在操作日志中有首次买入记录
- 如果某只基金在 `fund-holdings.md` 中有持有份额，但操作日志中没有任何该基金的交易记录 → 告知用户，请用户确认是否存在遗漏交易，或补充 PDF 导入

### 检查项2：持仓数据一致性检查

- 每次更新 `fund-holdings.md` 前，用操作日志重新推导所有基金的**持有份额**，与 `fund-holdings.md` 中记录的份额对比
- 如果发现差异（日志推导 ≠ 文件记录）→ 立即告知用户差异金额，并判断：
  - 差值 > 1 份 → 大概率是用户手动改过份额、或日志遗漏了交易 → 请用户确认
  - 差值 ≤ 1 份 → 舍入误差，直接以日志推导为准覆盖

### 检查项3：手动修改检测

- 读取 `fund-holdings.md` 后，如果 `数据基准日期` 以下的内容与 AI 上次写入时不一致（累计收益、持仓金额等字段被人改过）→ 以用户手动值为准，不覆盖；同时在简报开头标注："检测到你手动修改了持仓数据，AI将以你的数据为准，如数据产生偏差请联系确认"

### 检查项4：净值查询失败处理

- 查净值时如果天天基金网无法返回某基金某日数据 → 不跳过，在简报中告知用户具体是哪只基金、哪天数据缺失，并说明采用的替代方案（如使用最近一个可查到的净值）

### 检查项5：日志日期断层检查

- 导入新 PDF 或口述更新后，检查操作日志中是否有明显的日期断层（如连续2个月无任何交易记录，但基金仍在持有中）
- 发现断层 → 提醒用户："XX基金的交易记录从X月X日后空白，如果期间有未记录的买卖请告知"

## 数据更新流程

本项目支持两种更新方式：**导入蚂蚁财富导出的交易明细PDF**、以及**用户口述操作**。两种方式最终都只做两件事：更新本文件的"操作日志"，必要时同步更新 `fund-holdings.md` 的持有份额/持仓金额。

### A. PDF导入更新（蚂蚁财富交易明细）

**输入**：蚂蚁（杭州）基金销售有限公司导出的《基金（含资产管理计划）交易明细》PDF。

**核心字段（以PDF表头为准）**：

- 订单号
- 交易时间
- 交易类型
- 基金名称
- 基金代码
- 申请金额 / 申请份额
- 确认金额 / 确认份额
- 手续费
- 确认日期

**执行步骤**：

1. 打开PDF，确认标题为"基金交易明细"，并包含上述表头字段。
2. 读取PDF所有页面，提取表格数据：
   - 使用 `pdfplumber` 打开PDF，通过 `extract_tables()` 按表格边框识别行列结构。
   - 避免使用 `pypdf` 文本提取，后者对中文表格列的解析容易错位/截断。
3. 将每一条交易识别为一个"记录块"：
   - 以行首出现 `2026xxxx` 这类8位数字作为记录块起始（订单号的前缀段）。
   - 从该行起，持续合并后续文本，直到遇到下一个8位数字起始行，形成一条记录块。
4. 从记录块中解析字段：
   - 交易时间：识别被断行/拆分的时间，拼合成完整的交易时间。
   - 交易类型：匹配 `用户买入 / 用户卖出 / 用户认购 / 定投买入 / 用户跨TA转换 / 用户转换 / 机构分红`。
   - 基金代码：匹配6位数字。
   - 金额/份额/手续费：匹配两位小数的数值，按表头顺序对应到确认金额/确认份额/手续费。
   - 基金名称：从"交易类型"之后到"基金代码"之前的文本（去除换行、空格）还原。
5. 分类落库（写入本文件的"操作日志"）：
   - 转换：按"转出/转入"两条记录写入（PDF本身就是两行）。
   - 买入/认购/定投/卖出/分红：分别进入对应分类表格。
   - 同日多笔同基金交易：**不合并**，原样记录并在备注中标注（避免丢单）。
6. 排序与去重原则：
   - 默认按"日期时间"倒序排列。
   - 仅当订单号+交易时间+基金代码+确认金额完全一致时，才视为重复（一般不会自动去重）。
7. 校验：
   - 对照 PDF 的关键几笔操作（尤其是转换、卖出手续费）抽样核对字段没有错位。
   - 若 PDF 存在"申请份额/确认份额"缺失（常见为 `/`），对应字段记为 `-`，不臆造。

**同步更新 `fund-holdings.md` 的原则**：

- 交易明细PDF + 操作日志是最准确的确认数据（来自蚂蚁 + 基金注册登记机构）。
- 当操作日志已覆盖某只基金从**首次买入至今**的全部交易时，可直接从中推导当前持有份额并同步更新 `fund-holdings.md`。
- 推导方法：该基金所有买入（含认购、定投、转换转入）的确认份额之和，减去所有卖出（含赎回、转换转出）的确认份额之和 = 当前持有份额。
- 持仓金额 = 当前持有份额 × 最新净值（由AI在每日简报中查询后填入）。
- 例外：如存在分红再投资（份额增加但无买入记录的情况），需要确认分红方式是否为"红利再投资"，否则按现金分红处理（不影响份额）。

### B. 每日持仓数据更新（简报中自动执行）

**触发时机**：每次输出每日简报时，按以下步骤更新 `fund-holdings.md`。

**容错机制说明**：

- 由于可能非每日对话，更新时需判断"数据基准日期"与"最近一个完整交易日"之间的差距
- 如果间隔 N 个交易日 → 需依次查找其中每个交易日的净值，**按日累积计算**涨跌幅
- 如果无法查到中间某日净值 → 改为直接查最新日的净值，按份额×最新净值计算（舍弃逐日累积，但基准日期一并更新）

**执行步骤**：

1. 读取 `fund-holdings.md` 顶部的 `数据基准日期`，记为 D0。
2. 确定最近一个完整交易日 D_last（判断逻辑：当前时间是否已过 15:00 → 是则**当前日期**为 D_last，否则取**上一个交易日**）。
3. 若 D0 = D_last，说明已是最新数据 → 跳过更新。
4. 若 D0 < D_last，进入更新流程：
   a. 从天天基金网（fund.eastmoney.com）逐一查询每只基金在 D0→D_last 期间各交易日的**单位净值**。
   b. 累积计算涨跌幅后，得到最新持仓金额：持仓金额（新）= 持有份额 × 最新净值
   c. 更新累计收益：累计收益 = 持仓金额（新）+ 累计卖出到账 - 累计投入本金（累计投入/卖出从操作日志汇总，不变）
5. 写入后更新 `fund-holdings.md` 顶部的 `数据基准日期` 为 D_last。

**更新分工**：

- 持有份额：仅通过操作日志推导更新，每日净值更新不改份额。
- 持仓金额、累计收益：**AI自动计算并覆盖写入**，无需手动维护。如发现偏差，可手动修改；AI以手动值为准。

### C. 口述操作更新

每次操作后，按下面格式告知即可（AI会补齐到对应分类表格）：

- 日期时间：YYYY-MM-DD HH:mm（或说"今天10:58"也行）
- 操作：买入 / 卖出 / 认购 / 定投 / 转换转出 / 转换转入 / 分红
- 基金代码
- 确认金额（元）
- 确认份额（可选，但建议提供）
- 手续费（如有）
- 备注（可选，例如"跨TA""部分赎回""清仓"）
