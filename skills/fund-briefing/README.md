## 项目简介

**Fund Briefing** 是一个自包含的 AI Agent 技能包，将国际财经新闻分析与个人持仓基金顾问相结合。支持解析蚂蚁财富交易明细 PDF、维护操作日志、查询实时净值、生成结构化每日简报（国际时事 + 大盘分析 + 板块涨跌榜 + 持仓关联分析 + 反方论证 + 风险提示）。

两种运行模式：

- **有持仓模式** — 提供持仓总览、新闻事件与持仓基金关联分析、6级操作建议（含反方论证和量化情绪评分）。
- **无持仓模式** — 基于政策方向和大盘数据推荐关注赛道，不点名具体基金。

所有输出基于实时联网查询，杜绝编造内容、杜绝过期数据。

### 设计理念

本技能借鉴了开源多智能体交易框架 [TradingAgents](https://github.com/TauricResearch/TradingAgents)（Apache 2.0）的多个设计模式：

| 借鉴模式              | 在本技能中的实现                                                          |
| --------------------- | ------------------------------------------------------------------------- |
| Bull/Bear 辩论机制    | 简报中的"反方论证"——分析最可能出错的点，压制认知偏误                      |
| 决策反思 (Reflection) | 事件追踪中的"验证回填 + 反思总结"——预测→实际→教训闭环                     |
| 结构化评级体系        | 6级操作建议（加仓/小幅加仓/观望/小幅减仓/减仓/等待）+ 情绪量化评分 (0-10) |
| 全局新闻议程          | 8维度预定义搜索清单，确保每次简报覆盖全部关键信息面                       |

## 核心功能

| 功能              | 说明                                                            |
| ----------------- | --------------------------------------------------------------- |
| 蚂蚁财富 PDF 导入 | 解析蚂蚁财富《基金交易明细》PDF，自动生成结构化操作日志         |
| 每日财经简报      | 国际官方要闻 + A股/美股/港股市场分析 + 行业板块涨跌榜           |
| 持仓净值同步      | 从天天基金网自动查询最新净值，每日更新持仓金额和累计收益        |
| 持仓关联分析      | 新闻事件 → 利好的重仓股/细分赛道 → 具体持仓基金                 |
| 6级操作建议       | 🟢加仓 / 🟢➖小幅加仓 / 🟡观望 / 🟡🔴小幅减仓 / 🔴减仓 / ⏸️等待 |
| 情绪量化评分      | 影响方向附加 0-10 强度分 + 置信度（高/中/低）                   |
| 反方论证          | 分析最可能出错的点，平衡天然乐观/悲观偏差                       |
| 事件溯源追踪      | L3/L4 重大事件记录、预测→验证回填、偏差分析、可复用教训总结     |
| 市场数据归档      | 30天滚动每日数据 + 月度摘要压缩，支撑跨日趋势分析               |
| 用户画像          | 风险评级、赛道偏好、操作习惯，智能推断 + 用户确认               |
| 自动自检机制      | 覆盖所有已启用模块的完整性、一致性和连续行检查                  |
| 模块化设计        | 基金/股票/债券等资产类型按需启用                                |
| 持有期监控        | 自动追踪各基金7天惩罚费率到期日和30天免费赎回到期日             |
| 周日周报          | 每周日自动触发周度复盘，汇总汇率、大宗商品、三大市场指数波动    |
| 无持仓模式        | 基于政策和大盘数据推荐关注赛道，适合空仓用户                    |
| 隐私安全          | 用户数据与技能包分离，技能包可经 git 分发                       |

## 安装

1. 通过 `npx skills` 安装：

   ```bash
   npx skills add https://github.com/LovelyKein/skills
   ```

2. 安装 Python 依赖（仅 PDF 导入功能需要）：

   ```bash
   pip install pypdf
   ```

3. 首次激活时，技能会自动从 `templates/` 目录复制空白模板文件到工作目录，进入无持仓模式。

### 增强使用（可选）

在工作目录下创建 `AGENTS.md`（参考 `templates/agents-template.md`），可启用项目级增强：

- 用户身份与交易平台
- 按需启用股票/债券等其他持仓模块
- 项目特定约束与数据源偏好

不创建 AGENTS.md 时，技能仍可独立运行。

## 项目结构

```
fund-briefing/                              # 技能包（git 仓库，可分发）
├── SKILL.md                                # 技能定义文件（完整行为规范，自包含）
├── LICENSE.txt                             # MIT 许可证
├── README.md                               # 本文件
├── scripts/
│   └── parse_ant_fund_pdf.py               # 蚂蚁财富 PDF 交易明细解析脚本
└── templates/                              # 数据文件模板（空白，无硬编码用户数据）
    ├── fund-holdings-template.md           # 持仓数据模板
    ├── fund-operations-log-template.md     # 操作日志模板
    ├── market-archive-template.md          # 市场数据归档模板
    ├── user-profile-template.md            # 用户画像模板
    ├── event-tracker-template.md           # 事件溯源模板
```

运行时自动生成（首次使用从模板复制到工作目录）：

```
<工作目录>/                                 # 用户数据（不进 git）
├── AGENTS.md                               # 项目级增强（可选）
├── fund-holdings.md                        # 持仓数据（份额/金额/收益）
├── fund-operations-log.md                  # 操作流水 + 规则参考 + 自检机制
├── market-archive.md                       # 市场数据归档（30天滚动）
├── user-profile.md                         # 用户画像（风险/偏好/习惯）
├── event-tracker.md                        # 事件溯源与拐点追踪
└── fund-briefing/                          # 技能包（git 仓库）
```

## 环境依赖

| 依赖   | 版本   | 用途                              |
| ------ | ------ | --------------------------------- |
| Python | >= 3.8 | 脚本运行环境                      |
| pypdf  | >= 4.0 | PDF 文本提取（仅 PDF 导入时需要） |

数据来源均为公开网页（无需 API Key）：

- `fund.eastmoney.com` — 基金净值查询
- `data.eastmoney.com` — 大盘行情、板块资金流向
- `finance.sina.com.cn` — 美股、港股行情
- `news.cctv.com` / `xinhuanet.com` — 官方国际新闻
- `csindex.com.cn` — 指数行情、估值分位

## 参与贡献

欢迎提交 PR，请遵循以下规范：

1. **禁止硬编码数据** — 模板必须为空白，脚本不得嵌入任何用户专属数值。
2. **遵循文件命名约定** — 数据文件统一使用预定命名。
3. **SKILL.md 必须自包含** — 完整描述所有行为逻辑，不依赖外部文件。
4. **Python 脚本语法验证** — 提交前运行 `python -m py_compile scripts/parse_ant_fund_pdf.py` 确保通过。
5. **干净环境测试** — 在无数据文件的全新工作目录中验证首次运行初始化流程。

PR 流程：

1. Fork 本仓库
2. 创建功能分支
3. 修改并验证 `python -m py_compile scripts/parse_ant_fund_pdf.py`
4. 提交 PR 并附清晰说明
