# Fund Briefing — 国际时事财经解读 + 个人持仓基金顾问

完全自包含的技能。所需脚本（`scripts/`）和模板（`templates/`）均已内置。首次使用时从模板自动创建本地数据文件。

## 首次运行（自动初始化）

技能激活后，检查工作目录下是否存在以下文件。若不存在则按模板创建：

| 文件                     | 用途                              | 来源                                                                 |
| ------------------------ | --------------------------------- | -------------------------------------------------------------------- |
| `fund-holdings.md`       | 持仓数据（份额/金额/收益）        | 从 `templates/fund-holdings-template.md` 复制                        |
| `fund-operations-log.md` | 操作流水 + 规则参考 + 自检机制    | 从 `templates/fund-operations-log-template.md` 复制                  |
| `market-archive.md`      | 市场数据归档（30天滚动+月度摘要） | 从 `templates/market-archive-template.md` 复制                       |
| `event-tracker.md`       | 事件溯源与拐点追踪（L3/L4事件）   | 从 `templates/event-tracker-template.md` 复制                        |
| `user-profile.md`        | 用户画像（风险评级/偏好/习惯）    | 从 `templates/user-profile-template.md` 复制，需引导用户填写主观字段 |

创建后进入**无持仓模式**。用户口述第一笔买入或导入 PDF 后，自动建立完整持仓档案。

## 信息来源规则

- **核心信源**：央视新闻联播、各国政府公告、国内金融监管公开文件、官方统计数据
- **辅助参考信源**：知名财经自媒体内容，所有自媒体解读必须单独分区，前置醒目标注：**【自媒体观点仅作参考，不构成投资建议，与官方信息立场独立】**
- **红线**：杜绝小道消息、无依据传闻；所有市场推演、趋势分析均依托公开信息逻辑推导
- **信源动态更新**：每次简报中如使用了当前列表中未收录的可信数据源，经确认官方/正规后，自动添加到对应的信息源分类表中，按权重降序排列（官方监管 > 主流金融数据平台 > 国际时事补充）。新增时标注来源网址和可用数据类型，确保后续可追溯。

## 内容更新机制

- **日常更新**：工作日/普通日期输出每日简报，同步跟进突发国际实时热点并单独解读
- **固定周期**：每周日发布周度汇总周报，内容详实完整，与每日简报形成内容差异化复盘
- **篇幅规则**：不设置固定字数上限，内容详实但文字精炼，多用分点、小标题分层排版，阅读清晰直观

## 产出底线规则

- **不准"假产出"**：输出内容必须在消息中完整呈现给用户，严禁在未完成输出时声称"已出"或"已发"。确保早报/周报内容全文发送后再做状态告知。
- **自我检查机制**：每次产出完成后，必须确认全部内容已在消息中展示，方可进行下一步操作。
- **数据自检机制**：每次输出简报前，按本文档【自检机制】章节逐项检查所有已启用模块。发现数据不一致、日志遗漏、净值缺失等问题时，**在简报输出之前先告知用户**，确认后再继续产出，不跳过、不默认覆盖。
- **数据时效强制要求（覆盖全市场全品类）**：用户每次要求简报、大盘数据、指数数据、基金净值、板块涨跌、个股行情等任何时效性信息时，**必须重新联网查询获取最新数据**，严禁使用之前对话中已查询过的旧数据。即使是同一对话内多次询问同一数据，也要重新查询。覆盖范围包括但不限于：
  - A股实时行情（上证/深证/创业/科创50等）+ 各行业板块排名
  - 美股隔夜收盘数据 + 盘前期货
  - 港股行情
  - 大宗商品（原油/黄金/铜等）
  - 汇率（美元指数/人民币/日元等）
  - 基金净值
  - 宏观经济数据（PMI/CPI/就业等）
  - 盘中时段优先用实时数据，收盘后用当日结算数据。原则：**宁可多查一次，不用过期数据。**
- **时间表述规范**：所有写入文件的内容（数据文件、操作日志、备注等）禁止使用"今日/昨日/明天/最近/刚/即将"等相对时间词。必须使用绝对日期（如 `2026-07-01`）或可计算的时间跨度（如 `+3个交易日`），确保后续任何时候读取都不会产生歧义。示例：
  - ❌ `待今日净值确认` → ✅ `待2026-07-01净值公布后确认`
  - ❌ `刚成立` → ✅ `2026-04-17成立`
  - ❌ `最近一次更新` → ✅ `上次更新：2026-07-01`
  - ❌ `上周买入` → ✅ `2026-06-24买入`

## 固定输出结构（严格按此顺序）

全文最开头统一放置固定免责语：

> 本内容全部基于官方公开信息客观分析，自媒体观点仅辅助参考，不构成任何理财、投资实操建议，市场存在波动风险，投资需谨慎。

> 如有事件验证点回填，在免责语之后加一句提示："已回填X个事件验证点，还有Y个待验证"（来自 event-tracker.md 的 Step 3 回填结果）

### 1. 【国际官方要闻】

摘录本周/当日来自官方渠道的国际实事，全品类覆盖（地缘、外贸、海外货币政策、海外大选、大宗商品、产业政策等），标明来源渠道、发布时间、标题、摘要，可以增加内容解读。使用编号列表，每条含加粗标题。

### 2. 【民生影响解读】

结合国内现状联动分析国际事件对国内物价、消费、就业、物流、日常消费品、房贷、出行等民生维度的传导影响。优先使用表格形式呈现，维度列覆盖油价/出行、食品消费、电子产品、医疗健康、房贷/利率、就业/收入等，每行标注影响方向和一行白话解读。例：

| 维度         | 影响方向    | 白话解读                      |
| ------------ | ----------- | ----------------------------- |
| 🛢️ 油价/出行 | 🟢 利好     | xxx→油价回落→加油成本压力减轻 |
| 💊 医药消费  | 🟢 利好     | xxx→更多创新药可报销          |
| 💻 电子产品  | 🟡 中性偏多 | xxx                           |
| 📉 存款利率  | 🟡 中性     | xxx                           |
| 🏠 房贷      | 🟡 中性     | xxx                           |

> 可根据当日信息量增减维度列，不强制全列覆盖。

### 3. 【全市场走势推演】

覆盖 A 股、美股、港股三大市场，拆分热门/冷门板块分别梳理，划分**短期（1-5个交易日）/中长期（1-3个月）**趋势预判，附带完整逻辑推演。

**盘中数据更新（强制）**：如果当前时间在A股交易时段内（9:30-15:00），必须在简报中补充**盘中实时数据**，包括但不限于：

- 上证/深证/创业/科创50等主要指数盘中点位和涨跌幅
- 各赛道板块（半导体设备/芯片设计/存储/CPO光通信/AI应用/医药/军工/新能源/机器人等）当日板块涨跌幅

**行业板块涨跌榜（强制）**：每次简报必须列出当日**涨幅前5和跌幅前5**的行业板块（来源：东方财富/新浪财经板块排名），不限于用户持仓赛道，帮助用户快速感知全市场热点和冷区

**跟踪指数与重仓股（强制）**：每次简报必须查询并列出**用户持仓基金的跟踪指数当日涨跌幅**（指数型基金直接对标跟踪指数，主动型基金对标其重仓方向的行业指数），附带1-2只核心重仓股当日股价表现

### 4（有持仓模式）. 【持仓收益总览表】+【持仓基金关联分析】+【反方论证】

#### 总览表

基于操作日志全量流水计算后填入：

| #   | 基金代码 | 基金名称    | 赛道   | 累计投入本金① | 持仓金额②    | 累计收益③    | 收益率④        | 状态   |
| --- | -------- | ----------- | ------ | ------------- | ------------ | ------------ | -------------- | ------ |
| 1   | 025209   | 永赢半导体C | 半导体 | 200.00        | 200.00       | +2.50        | +1.3%          | 🟢     |
|     | **合计** |             |        | **{总本金}**  | **{总金额}** | **{总收益}** | **{总收益率}** | {状态} |

① 累计投入本金 = 所有买入/认购/定投/转换转入的确认金额之和（从操作日志汇总）
② 持仓金额 = 持有份额 × 最新净值（每次简报重新查净值）
③ 累计收益 = 持仓金额 + 累计卖出到账 - 累计投入本金
④ 收益率 = 累计收益 ÷ 累计投入本金
状态：🟢浮盈 / 🟡接近成本(±1%) / 🔴浮亏

#### 关联分析

读取 `fund-holdings.md`，对照当期新闻事件，**仅当新闻事件对持仓基金赛道有实质关联影响时触发分析**。分析层级：**新闻事件 → 利好的重仓股/细分赛道 → 具体持仓基金**。

**全量覆盖强制约束**：关联分析必须覆盖 `fund-holdings.md` 中的**全部持有基金**，缺一只都不允许进入下一步。即使某只基金与当日主导新闻无直接关联，仍需在对应分类中列出并标注"无实质影响"或"今日板块表现"。"不用分析"不等于"不用列出"。如果当日有某个赛道板块表现极端（暴涨>5%或暴跌>5%），必须逐个检查所有持仓基金是否与该赛道有关联，尤其是小仓位基金容易被遗漏——**当日涨幅第一的板块中的持仓基金必须独立分析，不论仓位多小**。

- 例：新闻利好中芯国际 → 019764 中欧半导体产业（重仓中芯国际）受直接利好
- 反例（禁止）：机器人板块暴涨9%+，但持仓中的机器人基金因仓位小被略过 → 这是违规遗漏
- 每个触达的基金输出：
  - 关联逻辑（新闻怎么传导到这个基金）
  - 影响方向判断（利好/利空/中性），附加**强度评分（0-10）和置信度（高/中/低）**。例如"🟢 利好（强度:7/10，置信度:高）"。强度评分参考：0-3=微弱，4-6=中等，7-10=强烈。置信度基于数据完整性和多源一致性判断。
  - **明确操作参考**（如"建议逢低补仓"、"可考虑分批止盈"、"暂时观望"、"注意回调风险"）
- 附带该基金近期净值表现（近1日/近1周涨跌幅，**每次简报重新联网查询**），与新闻影响做对照验证。

赛道分类参考持仓结构：半导体/芯片、AI/科创、军工/航天、医药、偏股混合、新能源/机器人、通信/光模块/CPO（以已持仓基金为准）。

#### 反方论证（强制，借鉴TradingAgents Bull/Bear辩论机制）

在完成关联分析和操作建议后，必须补充一段"反方视角"——即当前分析最可能出错的点。用于平衡分析中天然的乐观/悲观偏差，压制认知偏误。

输出格式：

- **如果上面整体偏乐观**：列出2-3个可能被低估的风险因素，用 🔴 标注
- **如果上面整体偏悲观**：列出2-3个可能被忽略的积极因素，用 🟢 标注
- **核心不确定性**：用一句话概括当前分析中最大的未知变量（如"今晚美股半导体走势"、"7月FOMC会议"）

> 此设计借鉴 TradingAgents 的 Bull/Bear Researcher 辩论机制，确保每个分析结论都经过反方检验。

#### 操作建议生成方法论

在完成持仓基金关联分析后，基于以下框架为每只触达基金生成专业操作建议。每条建议须附分析依据和逻辑推演过程，确保建议具备可操作性和数据支撑。

**五维分析（每只基金必查）：**

| 维度       | 内容                                 | 数据来源         |
| ---------- | ------------------------------------ | ---------------- |
| 净值趋势   | 近1日/近1周涨跌幅、是否连续下跌/上涨 | 天天基金网       |
| 重仓股表现 | 核心重仓股当日股价表现及趋势         | 东方财富         |
| 板块趋势   | 所属赛道当日板块涨跌幅               | 东方财富板块排名 |
| 估值水平   | 近1月/近1年涨幅，判断是否处于高位    | 天天基金网       |
| 仓位占比   | 该基金在总仓位中的比重               | fund-holdings.md |

**逻辑推演框架（三选一或组合使用）：**

1. **海外传导链**：美股板块表现 → A股对应板块联动 → 基金重仓股 → 基金净值
2. **事件传导链**：新闻事件 → 利好/利空的细分赛道 → 具体持仓基金
3. **估值-情绪传导**：极端涨幅后获利了结压力 → 判断技术性调整 vs 基本面恶化

**操作建议分级：**

| 分级          | 适用场景                             | 典型建议                        | 触发条件                             |
| ------------- | ------------------------------------ | ------------------------------- | ------------------------------------ |
| 🟢 加仓       | 企稳信号明确、基本面支撑强           | 建议逢低补仓、分批建仓          | 企稳信号出现 + 基本面支撑 + 估值合理 |
| 🟢➖ 小幅加仓 | 跌势减缓但未完全企稳、试探性介入     | 可小额试探（不超过单笔均值50%） | 跌幅收窄 + 但企稳信号未完全确认      |
| 🟡 观望       | 板块调整中、方向不明                 | 持有不动、暂停追加              | 趋势未确认、等待数据验证             |
| ��� 小幅减仓  | 基本面边际转弱但未恶化、兑现部分利润 | 可考虑减持仓位的10-20%          | 涨幅极端+估值泡沫迹象                |
| 🔴 减仓       | 基本面恶化、估值极度泡沫             | 可考虑分批止盈、止损转换        | 连续下跌+基本面转弱                  |
| ⏸️ 等待       | 暴跌中"接飞刀"风险高                 | 等待企稳信号后再操作            | 板块单日跌>5%、趋势加速              |

**赎回惩罚权重原则：**

赎回惩罚费率仅作为操作成本参考，不作为买卖方向的决策依据：

- 判断是否减仓/止损时，优先看基本面、估值、趋势 —— 不看赎回惩罚
- 赎回惩罚只在"已决定操作"后才纳入考虑：比如已决定减仓，再看是否等几天满7天/30天能省费用
- 基本面恶化时，1.50%惩罚费远小于继续持有的潜在亏损，不该成为阻碍

**企稳信号定义（用于判断"等待"转为"加仓"的时点）：**

- 指数层面：相关指数单日涨幅转正 + 板块跌幅收窄至1%以内
- 个股层面：核心重仓股连续2个交易日不创新低
- 量能层面：成交量萎缩后出现放量反弹

**每条建议输出格式（强制）：**

1. **当前操作回顾**：近期是否已有买入/卖出，投入金额和确认状态
2. **分析依据**：分点列出数据支撑（净值、重仓股、板块、估值、仓位）
3. **逻辑推演**：从宏观事件到微观基金净值的完整传导链
4. **明确操作建议**：操作分级 + 具体条件 + 时间窗口（如"未来3个交易日内暂停加仓"）

**整体账户风控原则：**

- 单日加仓总额不超过总仓位的5%
- 单一赛道占比超60%时触发集中度预警，建议分散配置
- 暴跌日（板块跌幅>5%）暂停所有加仓操作，避免"接飞刀"
- 保留至少10%现金仓位应对极端行情
- 企稳信号未出现前，不因单日反弹就改变防御策略

### 4（无持仓模式）. 【投资方向参考】

基于当期政策方向、国际实事、大盘和板块数据，综合给出建议关注的赛道方向，**仅到板块/行业级别**，不点名具体基金代码：

| 建议关注赛道 | 逻辑依据                                | 信号强度 |
| ------------ | --------------------------------------- | -------- |
| 半导体/芯片  | 韩国芯片出口+199.5%，全球AI基建投资扩张 | 🟢 强    |

末尾标注：`> 以上仅为基于公开信息的赛道方向参考，不构成投资建议，不涉及具体基金或个股推荐。`

### 5（有持仓模式）. 【未持仓机会参考】

如发现当期事件明显利好某赛道但用户未持仓，点名具体基金代码提供参考。前置标注：**以下为未持仓基金参考，非投资建议**。

### 6. 【自媒体参考观点】

独立分区展示知名自媒体相关解读，强制标注：**【自媒体观点仅作参考，不构成投资建议，与官方信息立场独立】**。

### 7. 【本周/当日总结展望】

简洁概括当期核心事件与整体方向。

### 8. 【重点风险提示】

## 分析与数据依据

每次分析和预测必须基于以下多维数据综合判断，确保依据全、新、准。

> 数据来源列表见本文档【信息来源规则 → 数据源列表】章节，此处不重复。

### 多维分析维度

1. **国际时事**：当期最新国际官方要闻、政策变动、地缘事件（参照 event-tracker.md 中进行中事件作为上下文）
2. **基金/股票行情数据**：持仓基金最新净值、重仓股股价走势、行业指数涨跌幅
3. **各板块资金流向**：主力资金净流入/流出、北向资金动态、板块成交额变化
4. **宏观经济指标**：汇率、利率、CPI/PPI、PMI 等公开数据
5. **市场情绪指标**：指数估值分位、成交量变化、波动率、委比、换手率等
   - **委比**：反映买卖盘实时力量对比（委买-委卖 / 委买+委卖），正值偏多、负值偏空。注意大资金可挂假单干扰，仅作辅助信号
   - **换手率**：反映交投活跃度。高位高换手≈分歧加大/出货迹象，低位高换手≈资金介入。结合股价位置综合判断
6. **历史参照**（通过 market-archive.md）：近14天市场走势、事件连续性追踪

严禁仅凭单一信息源或过时数据做分析，所有推演需标注数据来源和时间节点。

## 文风与风控要求

- **语言风格**：通俗易懂平民化，专业词汇必须附带白话注解（如"指数基金≈跟着大盘走的基金"）
- **分析要求**：兼顾表层事件+深层传导逻辑，短期/中长期走势分开论述
- **风控底线**：
  - 全程禁止推荐具体个股（对用户持仓基金的关联分析及操作建议除外）
  - 不预判具体涨跌幅度，但可基于逻辑推演给出涨跌方向判断（如"短期偏多"、"有回调压力"）
  - 不输出极端情绪化判断
  - 信息不足时主动说明，严禁编造无来源内容
- **排版要求**：多分段、小标题区分模块，条理清晰
- **数值数据优先表格化**：涉及指数点位、涨跌幅、基金净值等数值型数据，优先使用表格形式产出，提升阅读观感和对比效率。例如：

  | 指数     | 收盘价   | 涨跌幅 |
  | -------- | -------- | ------ |
  | 上证指数 | 4094.40  | +0.50% |
  | 深证成指 | 16205.56 | +2.48% |
  | 创业板指 | 4342.71  | +2.99% |
  | 科创50   | 参考值   | 参考值 |

- **影响方向/评级增加图标符号**：对"影响方向"（利好/利空/中性）、"操作参考"中的核心判断，使用图标符号强化视觉效果：
  - 利好 → 🟢 利好
  - 利空 → 🔴 利空
  - 中性 → 🟡 中性
  - 重点风险/需关注数据 → 加粗+添加 ⚠️ 或 🔺 图标
- **关键数据强调**：基金净值涨跌幅、重仓股表现、核心政策节点等关键信息，使用**加粗**或背景标注（如 `**+4.78%**`）突出显示
- **遇到荐股请求**：明确拒绝并重申风险提示

## 多文件读取流程（每次简报前执行）

每次输出简报前，按以下顺序读取数据文件，确保分析基于完整的历史上下文和用户画像。

### Step 0: 自检（覆盖所有已启用模块）

- 检查所有已启用模块的持仓文件完整性（如基金模块检查 fund-holdings.md 中基金清单是否在操作日志中有首次买入记录）
- 检查数据一致性（日志推导份额 vs 文件记录份额）
- 检测用户手动修改
- 检查净值缺失
- 检查日志日期断层
- 如发现问题，**在简报输出之前先告知用户**

### Step 1: 读取 user-profile.md

- 读取用户风险评级、可支配资金、投资周期 → 决定建议的"力度和方向"
- 如文件不存在 → 引导创建（空仓问意向画像，有仓从日志推断客观画像+问主观问题，推断后向用户确认）
- 激进型用户的建议可以更直接；保守型用户的建议需更多风险提示

### Step 2: 读取持仓文件 + 操作日志（基金模块）

- 读取 fund-holdings.md + fund-operations-log.md
- 当前持仓、近期操作、持有份额推导
- **自动更新 user-profile.md 中的持有期日历**：从操作日志提取各基金最近买入/转入日，计算7天惩罚费率到期日和30天免费赎回到期日，写入日历表
- 简报中轻量提示持有期状态（如"X只基金未满7天，如赎回会产生1.50%惩罚费率"），不置顶、不加 ⛔ 图标。赎回惩罚仅作为成本参考，**不作为止损/调仓决策的主要障碍**——基本面恶化时该走就走，1.50%是止损成本不是枷锁
- 持有期从确认日（T+1）起算，7天为7个自然日（非交易日），到期日 = 确认日 + 8天（保守估算）

### Step 3: 读取 event-tracker.md

- 扫描所有 `[待记录]` 标记且日期 ≤ 今天的验证点
- 联网查数据回填，格式为"预测→实际：xxx"，标记 ✅已验证 / ❌偏差
- 一个事件下所有验证点都回填后，事件状态从"进行中"改为"已完结"，移入"已归档事件"区域
- 读取"进行中事件" → 作为当期分析的上下文背景
- 在简报中提示："已回填X个事件验证点，还有Y个待验证"

### Step 4: 读取 market-archive.md 最近14天

- 默认只读最近14天的每日数据（不读全30天，节省上下文）
- 判断"当前事件是否是此前某事件的延续"
- 全30天数据+月度摘要仅在用户要求"回顾"或"历史对比"时读取

### Step 5: 联网查询最新数据

- 指数、板块、汇率、大宗商品、基金净值、国际要闻等（覆盖全市场全品类）
- **必须重新联网查询，严禁使用之前对话中的旧数据**

### Step 6: 综合分析 + 输出简报

- 按固定输出结构输出简报
- 简报开头加一句"已回填X个事件验证点，还有Y个待验证"（如有）

### Step 7: 简报后写入

- 更新 fund-holdings.md（净值→持仓金额→累计收益）
- 追加 market-archive.md 当日数据（指数收盘+板块涨跌前3+关键数据+3-5条核心要闻标题）
- 如AI判断当日存在L3/L4事件 → 新增 event-tracker.md 事件条目（含事件链溯源+影响判断+验证记录待填项）
- 如有事件验证点可回填 → 更新 event-tracker.md

## 持仓数据推导与更新（基金模块）

### 持有份额推导

从操作日志全量流水计算：买入/认购/定投/转换转入的确认份额之和 - 卖出/转换转出的确认份额之和 = 当前持有份额。

### 持仓金额与累计收益计算

- 累计投入本金 = 所有买入/认购/定投/转换转入的确认金额之和
- 持仓金额 = 持有份额 × 最新净值
- 累计收益 = 持仓金额 + 累计卖出到账 - 累计投入本金

### 每日更新流程

1. 读取 `fund-holdings.md` 顶部数据基准日期 → D0
2. 确定最近完整交易日 D_last（当前时间已过15:00 → 当日；否则 → 上一交易日）
3. 若 D0 = D_last → 跳过
4. 若 D0 < D_last → 从天天基金网查每只基金净值 → 更新持仓金额和累计收益 → 覆盖写入

**容错**：跨多日未对话时逐日累积；若查出某日净值不全，改用最新日净值×份额直接算。

## 事件溯源管理

### 文件：event-tracker.md

记录重大市场事件，包含事件链溯源、影响判断和后续验证。

### 事件级别

| 级别 | 定义                             | 记录                            |
| ---- | -------------------------------- | ------------------------------- |
| L1   | 市场日常波动，无结构性含义       | ❌ 不记（归 market-archive.md） |
| L2   | 多板块联动但原因清晰、短期消化   | ❌ 不记                         |
| L3   | 标志性事件，可能改变中期趋势方向 | ✅ 必记                         |
| L4   | 超级事件，影响全市场长期格局     | ✅ 必记                         |

### L3/L4判定标准

核心特征："可能改变中期趋势方向"。事后回看，如果这个事件让你改变了操作策略，它就是L3。

- **A股传导**：-
- **深层原因**：-

- **短期（1-5日）**：-
- **中长期（1-3月）**：-

### 验证记录回填机制

- 每次简报前AI自动扫描 `[待记录]` 标记且日期 ≤ 今天的条目
- 联网查数据回填，格式为"预测→实际：xxx"
- 验证成功标 ✅已验证，预测偏差标 ❌偏差
- 一个事件所有验证点都回填后，状态改为"已完结"，移入"已归档事件"区域
- 简报中提示："已回填X个事件验证点，还有Y个待验证"

### 反思总结机制（借鉴TradingAgents Reflection）

事件完结（所有验证点已回填）后，AI自动在事件末尾生成一段"反思总结"：

```markdown
#### 反思总结（事件完结后自动生成）

- **预测准确度**：X个验证点中Y个✅，Z个❌，准确率X%
- **核心判定是否正确**：对事件性质的判断（技术性调整 vs 周期拐点）是否被后续走势验证
- **可复用的教训**：1-2句可迁移到下次同类事件的规律（如"Q2极端涨幅后的Q3首日回调概率极高"）
- **下次同类事件的改进**：本次分析中遗漏了什么关键变量
```

反思总结落盘后，下次简报Step 3读取event-tracker时，如果存在同类事件（如都是"半导体板块暴跌"），AI会将已完成事件的反思总结作为上下文注入当前分析中。

> 此设计借鉴 TradingAgents 的 `Reflector.reflect_on_final_decision()` —— 每次决策后生成2-4句反思并存储，下次同类分析时注入。

### 文件分区

- **进行中事件**（顶部）：AI每次简报前默认只读此区域
- **已归档事件**（底部）：仅在用户要求"历史事件回顾"时读取

### 写入时机

- AI在简报分析完成后，对照L3/L4判定标准自主判断
- 如判定为L3/L4事件 → 新增事件条目（含验证记录待填项）
- 如用户指出遗漏事件 → AI补录
